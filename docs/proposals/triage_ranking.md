2. **AI 責任邊界**: 醫療/勞動補助/族群代理/法律建議全部踩線. 需統一加 disclaimer + 明確"AI為初篩, 非最終認定" + deterministic 規則白名單兜底.
3. **雙北對稱包裝**: 多提案在資料 ID 列表偽裝雙北平等, 實際新北側資料缺失. 應改"資料覆蓋透明聲明", 把缺口本身做成 insight.
4. **32 hr 可行性樂觀**: MVP scope 普遍超載. 統一砍 30% feature → Day1 ship 1 個雙北組件 + 1 地圖, 其餘 Day2 補.
5. **30 RPM 並發保險缺**: AI 用法多預期高頻互動. 必須前端 cache + batch 預生成 + rate limit fallback to template.
6. **大螢幕「無人輸入」模式**: 多提案是互動式 UI, 但公共大螢幕無人操作. 需「自動輪播 + 即時資料 hook」模式設計.
7. **行動入口都是 mock**: 「點此通報」「點此預約」「點此申請」全是 hard-code 連結. 至少做 1 個提案有真實深度行動 (e.g. deep-link 到 1999/政府 API).
