## 提案 #1: 跨橋通勤可靠度指數 (Cross-Bridge Commute Reliability Index, CBCRI)

**Pitch (一句話)**: 把「從新北家到台北上班今天會遲到嗎？」變成像氣象指數一樣的每日「通勤可靠度天氣計」，並用 AI 解釋為什麼今天紅燈、給出唯一一條替代路徑。

**核心受眾 (Persona)**:
- 主受眾：住三重/蘆洲的機車族 (35 歲，每日 8:00 騎中興橋進中山區上班)、住板橋的捷運族 (28 歲，每日板南線到市府站)、住中和的雙北通勤公車族 (50 歲，搭 235/橘 5 過華中橋)。
- 次受眾：交通局決策者 (橋梁壅塞與替代道分流的政策評估)。

**雙北痛點**:
1. 雙北以「橋」為界，**14 座主要聯外橋梁** (中興橋、忠孝橋、台北橋、華江橋、華中橋、光復橋、中正橋、福和橋、永福橋、秀朗橋、北安、社子、麥帥、堤頂大道) 是日通勤瓶頸 — 任一橋出事，全城堵。
2. 現有官方 dashboard (見 `official-dashboard-inventory.md`) 有「交通路況」與「最新交通事故月統計」等獨立組件，**但沒有把「橋」當成主軸的整合視圖**，使用者無法回答最切身的問題：「今天從我家過哪座橋最快？」
3. 雙北落差：台北市內 VD 偵測器密集 (`/Road/Traffic/VD/City/Taipei`)，新北側橋頭多為事故黑點但偵測點稀疏 — 凸顯資源不對稱。

**核心價值 (看完能做什麼)**:
- 早上出門前 1 個畫面看到自家所在行政區「過橋可靠度」紅綠燈與預估通勤時間。
- AI 用一句話告訴你「今天紅燈：因為中興橋 7:42 有事故；建議改走台北橋 (+ 6 分鐘) 或改捷運 (+ 12 分鐘但 90% 機率不誤點)」。
- 政府視角：歷史 VD + 事故疊圖找出「結構性瓶頸 vs 偶發性瓶頸」差異。

**Demo 衝擊力 (大螢幕公共場合 2 分鐘腳本骨架)**:
1. **[0:00-0:20] Hook** — 大螢幕顯示一張雙北地圖，14 座橋全閃紅。畫面中央彈出「今天平均跨橋通勤多花 17 分鐘 = 一年 68 小時 = 多上班 8.5 天」。
2. **[0:20-1:00] 切換組件** — 點選「中興橋」→ 路況圖層、即時車速折線、橋頭事故紅點、替代橋的旅行時間對比，全部聯動高亮。下拉切「台北 / 雙北」展示資料覆蓋差異。
3. **[1:00-1:40] AI 演示** — 自然語言輸入「我住三重，9 點前要到中山國中站」→ AI 用 RAG 檢索當下 VD/事故/Alert，生成「中興橋 → 改走台北橋；預估到站 8:54；信心度 82%」。
4. **[1:40-2:00] 行動入口** — 一鍵「加入 Google Maps 導航」+「訂閱我的橋通勤通報」 (LINE Notify 推播；Demo 用 mock)。

**雙北對比角度**:
- 同一座橋兩端：北端 (台北) VD 密集、南端 (新北) 偵測點少 → 揭露「資料量不均」是整合分析的真痛點。
- 14 橋按「壅塞嚴重度」排名：板橋光復橋與中和秀朗橋常年穩居前 3，台北端則是中興橋 — 政府可量化「該補哪座橋的疏導預算」。

**AI 應用 (選用)**:
- 場景: 自然語言 → tool calling 路徑解釋。User 輸入「我從蘆洲到內湖科學園區」，模型呼叫 4 個 tool: `get_section_status(bridges=...)` → `get_alerts(area=...)` → `get_bus_eta(stops=...)` → `compose_recommendation(...)`，最後 llama3.3-ffm-70b 生成兩段中文：(a) 路線推薦含預估時間信心度、(b) 紅燈成因 (例：忠孝橋 8:00 事故影響到 9:30)。
- 30 RPM 限制：每位使用者 query 預估觸發 1 次 LLM 呼叫，黑客松 demo 場合單機 demo 完全充裕；正式場景需做 token-cache + 指紋去重。

**可解釋性**:
- 模型不直接決定路線；路線是規則引擎 + MaaS API 算出，**llama 只負責「翻譯」與「敘事」**。畫面同步顯示「這句話依據哪些原始 API 回應」(展開可看 JSON)，避免黑箱。
- Reliability Index 公式公開: `0.5 * 速度偏離歷史平均z-score + 0.3 * 事件影響半徑 + 0.2 * 替代橋備援度`。

**行動入口**:
1. 一鍵深連結 Google Maps / Apple Maps 導航。

**與現有 dashboard 差異**:
- 既有 dashboard 有「交通路況」與「最新交通事故月統計」**獨立組件**；本提案以「橋」為敘事主軸把上述資料 **空間 join + 時序疊加 + AI 敘事**，產出單一可決策視圖。
- 既有並未呈現「替代橋對比矩陣」與「Reliability Index」，這是新組件。
- 既有「公車車上擁擠度」未與橋梁對應；本提案做跨橋公車路線過濾 (235/橘 5/513/615 等聯外路線)。

**資料品質風險**:
- 新北 VD 點少 → 以「橋頭」就近匹配；不足者用 ETag 補。
- A1 統計為月資料 → 用於熱區疊圖而非即時；即時事故由 `/v2/Road/Alert` 取代。
