## UI 組件設計（4 組件）

---

### 組件 1：文化共融熱點地圖（含地圖圖層）

**類型：** 互動式地圖（Leaflet.js / Mapbox）

**功能描述：**
以台北市與新北市為範圍，在地圖上疊加多個資料圖層：
- **圖層一 — 藝文空間**：新北市文化局藝文空間、客家文化園區、原住民場館，以不同顏色區分場館類型
- **圖層二 — 活動熱力圖**：依文化快遞及藝文活動的地址，疊加熱力圖展示文化活動密集程度
- **圖層三 — 族群服務據點**：原住民家庭服務中心、新住民社區關懷據點

**互動設計：**
- 圖層開關切換面板（右上角）
- 點擊場館彈出資訊卡（名稱、類型、近期活動）
- 搜尋框：依族群類型或活動類型篩選

**資料來源：** 新北市政府文化局藝文空間一覽表、臺北市原住民場館介紹、臺北市政府文化局文化快遞資訊、新北市政府文化局藝文活動

---

### 組件 2：多元族群文化活動時間軸

**類型：** 互動式時間軸 + 篩選面板

**功能描述：**
整合台北市與新北市的文化活動資料，依時間軸展示各族群文化活動的分布，讓使用者掌握近期文化事件：

- 上方**篩選條**：依族群分類（客家 / 原住民 / 新住民 / 一般市民）、地區（台北 / 新北）篩選
- 中央**時間軸**：橫向滑動顯示每週活動排程，活動卡片以族群色碼標示
- 下方**活動卡片列表**：顯示活動名稱、日期、地點、主辦單位，點擊連結至詳細資訊

**視覺設計：**
- 客家活動：橘紅色標記
- 原住民活動：深綠色標記
- 新住民活動：藍紫色標記
- 一般藝文活動：灰藍色標記

**資料來源：** 臺北市政府文化局文化快遞資訊、新北市政府文化局藝文活動、新北客家文化活動、臺北市補助原住民民俗文化及社會教育活動

---

### 組件 3：觀光資源與遊客趨勢分析

**類型：** 複合圖表儀表板（折線圖 + 長條圖 + 排行榜）

**功能描述：**
呈現雙北主要觀光景點的遊客流量趨勢與文化資源分布的量化分析：

- **左側 — 景點遊客趨勢折線圖**：台北市主要觀光景點遊客數（按年/月）趨勢，可選擇特定景點比較
- **中央 — 景點遊客人次排行榜**：新北市景點遊客人次統計，Top 10 橫向長條圖
- **右側 — 文化補助資源分佈圓餅圖**：依族群別（客家/原住民/一般藝文）展示補助金額與件數佔比
- **底部 KPI 卡片**：總景點數、本月文化活動場次、觀光客總人次、文化補助總金額

**互動設計：**
- 年份/季度 Slider 控制時間範圍

**資料來源：** 臺北市主要觀光景點遊客數統計、新北市景點遊客人次統計、臺北市客家文化活動補助申請情形、臺北市政府文化局藝文補助案

---

### 組件 4：多元族群融合指數儀表板

**類型：** 族群分布雷達圖 + 人口統計卡片

**功能描述：**
以數據呈現雙北城市多元族群的社會融合現況，作為政策制定與文化推廣的參考指標：

- **上方 — 族群組成概況卡**：依區域（台北市各區 / 新北市各區）顯示原住民族人口數、新住民人口數及各族別比例，可依行政區切換
- **中央 — 族群文化參與雷達圖**：原住民族、客家族群、新住民在文化活動、教育資源、社福服務三個面向的參與度對比
- **下方 — 族群別人口趨勢折線圖**：現住原住民族種族別人口數（近5年趨勢），顯示各族群人口消長
- **側邊 — 融合亮點區塊**：標示雙北市在族群融合方面的政策亮點（如客家文化研習班開設場數、原住民部落大學課程人數等）

**資料來源：** 現住原住民族種族別人口數、臺北市原住民人口數按民族別分、新北市各區原住民人數統計表、新北市原住民族別人數排行榜統計表、原住民族概況

## 技術建議

| 面向 | 建議技術 |
|------|---------|
| 地圖渲染 | Leaflet.js + OpenStreetMap / Mapbox GL JS |
| 圖表庫 | Plotly.js 或 ECharts（支援熱力圖、雷達圖） |
| 前端框架 | React + Tailwind CSS |
| 資料整合 | 前端直接 fetch 開放資料 API（台北 / 新北） |
| 多語言景點支援 | 切換中/英文 API 資料，對應 i18n |

---

## 預期效益

1. **促進文化參與**：市民可即時掌握雙北各族群文化活動，降低參與門檻
2. **提升城市形象**：多語言景點資料與地圖可服務國際旅客，強化觀光競爭力
3. **輔助政策決策**：族群融合指數儀表板提供量化依據，協助文化局資源配置
4. **強化社會融合**：跨族群活動時間軸促進市民對多元文化的認識與參與
