> 調研日期：2026-04-17
> 資料來源：data.taipei, data.ntpc.gov.tw
> 調研目標：找出未在官方儀表板實作的通勤相關資料集

## 已跳過（官方已實作）

- 交通路況、公共停車場、今日施工資訊、最新交通事故月統計、監視器分佈

- 電動巴士比例、自行車道路統計資料
- 捷運系統全系列（人流趨勢、各線擁擠程度）
- 自行車道、公車站、人行道分布（地圖圖層）

| 項目 | 內容 |
|------|------|
| 來源平台 | data.taipei |
| 資料集 URL | https://data.taipei/dataset/detail?id=b5aaf33a-a6dc-4836-bce6-09986241fe11 |
| 更新頻率 | 即時（每分鐘）—實際資料檔 Last-Modified 顯示當日即時更新 |
| 資料格式 | XML（GZ 壓縮），API 介接 |
| 有無地理欄位 | ✓ 有（StartWgsX / StartWgsY / EndWgsX / EndWgsY，路段起終點 WGS84 座標）|
| 雙北支援 | △ 僅臺北（臺北市交通控制中心 VD 偵測器資料）|

**關鍵欄位：**
- `SectionId`：路段代碼
- `SectionName`：路段名稱（如「承德路 大業路-吉利街」）
- `AvgSpd`：平均速率（km/h）
- `AvgOcc`：偵測器佔有率（%）
- `TotalVol`：總流量（輛次）
- `MOELevel`：服務水準等級（0=暢通、1=緩慢、2=壅塞）
- `StartWgsX/Y`、`EndWgsX/Y`：路段起終點座標

**實際資料端點：** `https://tcgbusfs.blob.core.windows.net/blobtisv/GetVD.xml.gz`
（使用量高達 31,496,747 次，是平台最高使用量資料集之一）

**應用場景：**
即時呈現全臺北市各路段車速與服務水準，協助市民辨別壅塞路段、輔助政府進行交通疏導決策。

**組件設計草稿：**
- 圖表類型：道路熱力地圖（依 MOELevel 或 AvgSpd 上色）
- 城市切換：✗ 不支援（僅臺北）
- 地圖圖層：✓ 有（線段，路段起終點連線）
- 互動功能：下拉選擇查看路段詳情、時間軸回放（需搭配歷史資料）

### 組合方案 A：雙北道路即時動態儀表板

- 技術亮點：WGS84 路段座標直接繪製熱力地圖，MOELevel 直接對應壅塞顏色

### 組合方案 B：雙北公車智慧通勤地圖
- 核心資料：臺北市公車路線軌跡 + 新北市公車站位資訊 + 新北市公車預估到站時間
- 功能：地圖顯示公車路線軌跡，點選路線查看各站到站時間，即時刷新
- 技術亮點：雙北資料完整、有地理欄位、更新頻率高

### 組合方案 C：YouBike OD 流動分析
- 核心資料：臺北市YouBike起訖站點統計（GeoJSON OD矩陣）
- 功能：以流線圖呈現 20,680 條 OD 對，篩選週間/週末、起訖行政區
- 技術亮點：資料已含 LineString geometry，直接匯入地圖，視覺衝擊力強

### 組合方案 D：新北市即時停車整合
- 核心資料：新北市公有路外停車場即時剩餘車位 + 新北市路邊停車空位
