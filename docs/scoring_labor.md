## 構想細節評價

### L-01-1：複查優先佇列引擎
**綜合 8.2（A）— 主題首選之一**
**雙北必要性 10 分**：台北 15,007 筆違規以薪資/退休金/工時為主（服務業）vs 新北 14,155 筆以機械設備/特殊危害為主（製造業），同樣排序邏輯在新北必須大幅提高「職安型違規」權重。資料規模對等且公開，下拉切換邏輯天然成立。**設計亮點**：滑桿讓主管調整行業風險權重（人機協作），把「係數主觀」轉成「係數可調」是教科書級的死穴反擊。**核心風險**：須準備行業期望值基線資料（GCIS 商工登記取代理），24h 內可完成。

### L-01-2：違規模式 × 地理聚集預警
**綜合 7.6（A）**
新北工業區（中和、土城、三重、板橋）的同業聚集 vs 台北商業街型聚集，兩種地理模式本身是政策亮點。**核心風險**：聚集顯著性判定可降級為「同郵遞區號 × 同行業 × 同月 COUNT 比值」簡化版，不必上完整空間統計。**建議**：作為 L-01-1 的延伸視覺切片（地圖呈現），共用資料管線。

### L-02-1：失業熱點預警地圖（三信號同步觸發）
**綜合 8.1（A）— 強推**
「比失業潮早 3 個月看到」的價值主張極強，符合 Phase 3「從呈現 → 洞察 → 行動」願景。台北服務業 vs 新北製造業的政策回應差異（數位技能 vs 跨業轉職）讓雙北並排呈現有政策意義。公司登記、勞保人數、求職人數三項都是雙北月別公開資料。**核心風險**：閾值主觀性可用「歷史已知失業潮反向驗證」防守；「假陽性多開一班職訓 vs 假陰性錯失預警」的非對稱風險論述極具說服力。

### L-02-2：就服站預載壓力指數
**綜合 6.9（B）**
站長 + 局長雙端使用情境清晰，但「失業信號 → 就服站流量」的轉換模型需設計，24h 內只能做簡化版（按行政區人口比例分配）。**致命質疑**「失業者多直接上 104 不去就服站」需主動限縮使用者為「需政府介入的弱勢族群」。**建議**：作為 L-02-1 的下游延伸面板。

### L-03-1：移工職安風險矩陣
**綜合 7.7（A）— 強推**
**全國平台不敢做的跨部會整合**：移民署仲介評鑑 + 勞動部職災 + 違反就服法。台北看護工/家事移工（過勞、人身安全）vs 新北製造業移工（機械、化學）兩類仲介評鑑指標應不同——這個發現本身是政策洞見。**核心風險**：(1) 職災通報偏差（高風險仲介可能少報）需主動說明限制；(2) 違反就服法和評鑑等第是獨立驗證維度，多源校正邏輯要在 demo 中清楚呈現。

### L-03-2：看護移工投訴熱區識別
**綜合 6.5（B）**
「沉默地區 = 移工不敢投訴」推理太軟。投訴資料取得需配合機關，非完全公開。**建議**：把「沉默高風險區」定位為「待驗證假說 + 主動外展跟進」的閉環工具，作為 L-03-1 的延伸面板。

### L-04-1：就業資源可及性缺口地圖
**綜合 8.0（A）— 強推**
**「台北高齡認知障礙 vs 新北偏遠地理障礙」是雙北必要性最強的差異論證**——同樣的低就業參與率，背後成因完全不同，政策建議分兩層（地理新設點 vs 外展訪視）。市民有感度高（弱勢族群直接受益）。低收入戶、就服站點位、村里 GeoJSON 都是公開資料。**致命質疑「地理近 ≠ 使用率高」的兩層建議反擊**極具系統性思考深度。

### L-04-2：弱勢就業個案外展優先排序
**綜合 6.5（B）— 個資死穴**
個案資料涉個資，必用匿名化假資料。「就服員早就知道誰需要外展」的質疑可用「個人知識 vs 機構知識」（人員異動無縫交接）反擊，但說服力中等。**建議**：作為 L-04-1 的延伸場景描述（demo 用 mock 資料展示流程），不獨立。

### L-05-1：職安三維風險熱圖
**綜合 8.2（A）— 主題首選之一**
**雙北必要性 10 分**：新北製造業「高職災 + 高停工 + 高違規」三重疊 vs 台北服務業「高違規但低職災（過勞型職病難認定）」+ 大型工程「高停工」——兩城呈現完全不同的職安樣態，並排展示說明力極強。三個資料集都是雙北公開資料，**正規化（每百家業者中有幾家）**是 SQL 做不到的真實 AI 角色。**致命質疑「職災通報率」的反擊**：停工 + 違規兩維度通報偏差較小，三維互補可校正。

### L-05-2：大型工程職安生命週期
**綜合 6.7（B）**
時間軸視角是創意切角，但「停工原因有很多種」的假陽性問題真實。**建議**：作為 L-05-1 的延伸（點擊熱圖某行業 → 展開該行業典型工程的生命週期），共用資料。

### 黑客松四組件組合建議
本主題 A 級密度最高，建議主開發 **L-01-1 + L-02-1 + L-05-1 + L-04-1**：

- **L-04-1 就業資源可及性**（含里級熱圖 + 就服站點 + 新設點推薦）→ 滿足「至少 1 個地圖組件」
- **L-02-1 失業熱點預警地圖**（含地圖，三信號同步觸發行政區色階）→ 第二個地圖加分
- **L-05-1 職安三維風險熱圖**（行業 × 地區色塊矩陣 + 停工即時觸發）
- **L-01-1 複查優先佇列**（排名列表 + AI 風險解釋 + 權重滑桿）

四個組件涵蓋勞檢 / 失業預警 / 職安 / 弱勢就業四大向度，每個構想都有「雙北產業結構差異 → 兩種權重邏輯/政策建議」的對比論述，雙北切換不只是換資料，而是換解讀框架——這是「應用 40%」的核心加分點。

### 主題優勢
- **A 級構想數量最多（6 個）**，組件選擇彈性高
- **雙北必要性論證最強的主題**（產業結構差異是台北 vs 新北最鮮明對比）
- 與 official-dashboard-inventory.md 完全不重疊（既有儀表板無勞動就業組件）
- 開放資料完整：違規名單、職災、停工、勞保、公司登記、求職人數均雙北公開

### 主題風險
- 個資取得困難（L-04-2 個案、L-03-2 投訴、L-04-2 個資）→ 多個構想需用 mock 資料 demo
- 行業期望值基線需自行建立（GCIS 商工登記分母），需提前準備
- 「政府端工具」市民有感度普遍中等，需在 demo 強調勞工福祉間接受益

### 強烈建議
**若主題選擇陷入糾結，本主題 + 韌性防災是最值得首選的兩個主題**：
- 韌性防災：雙北必要性自然（流域、消防互助）+ 政府/市民雙端有感
- 勞動福祉：A 級構想最多 + 雙北產業差異論證最強 + 與既有儀表板不重疊
