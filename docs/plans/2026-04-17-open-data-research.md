# 雙北黑客松開放資料調研計劃

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 從 data.taipei 與 data.ntpc.gov.tw 系統性地挖掘六大主題的高價值開放資料集，每個主題找 10+ 個，並為每個資料集設計具體的組件視覺化方案，輸出供後續黑客松開發直接使用的研究文件。

**Architecture:** 使用 Playwright CLI 逐一瀏覽兩個資料平台的搜尋結果頁，針對六大主題各自搜尋關鍵字，過濾掉 `docs/official-dashboard-inventory.md` 中已實作的資料，對每筆候選資料集做深度記錄（欄位、更新頻率、地理屬性、圖表設計草稿），最終輸出主題研究文件與精選排名。

**Tech Stack:** playwright-cli（瀏覽器自動化）、Markdown 記錄文件

**重要參考文件：**
- `docs/official-dashboard-inventory.md` — 已實作組件清單（避免重複）
- `docs/2026雙北程式設計節競賽工作坊 競賽規則簡報.md` — 競賽規則與評分
- `docs/open-data-research/` — 本計劃所有輸出目錄

**資料來源：**
- 臺北市：https://data.taipei/
- 新北市：https://data.ntpc.gov.tw/

**競賽核心需求（時刻牢記）：**
- 至少 4 個雙北組件（可切換臺北市/雙北的資料）
- 至少 1 個雙北組件含地圖圖層
- 資料必須使用開放資料（來源合法、可驗證）
- 不得套用現有範例組件

---

## 每份主題研究文件的格式規範

每個主題輸出一份 `docs/open-data-research/theme-XX-<name>.md`，每個資料集記錄格式如下：

```markdown
### [資料集名稱]

| 項目 | 內容 |
|------|------|
| 來源平台 | data.taipei / data.ntpc.gov.tw |
| 資料集 URL | https://... |
| 資料集 ID/識別碼 | （如有）|
| 更新頻率 | 每日/每月/不定期... |
| 資料格式 | CSV / JSON / API |
| 有無地理欄位 | ✓ 有（欄位名：lat/lng/WKT...）/ ✗ 無 |
| 雙北支援 | ✓ 臺北＋新北 / △ 僅臺北 / △ 僅新北 |
| 官方已實作 | ✗ 未實作 |

**關鍵欄位：**
- `欄位名`：說明

**應用場景：**
（一句話說明這份資料能解決什麼問題）

**組件設計草稿：**
- 圖表類型：（e.g. 縱向長條圖、行政區熱力圖）
- 城市切換：✓ 支援 / ✗ 不支援
- 地圖圖層：✓ 有（點位/線段/多邊形）/ ✗ 無
- 互動功能：（e.g. 下拉選單選擇年份/類別）
- 資料對應：（x 軸/y 軸/顏色代表什麼）
- 黑客松主題契合度：⭐⭐⭐⭐⭐ / ⭐⭐⭐⭐ / ⭐⭐⭐
```

## Task 1：主題01 — 智慧通勤

**輸出檔案：** `docs/open-data-research/theme-01-transport.md`

**搜尋關鍵字（臺北）：** 公車、客運、計程車、自行車、機車、步行、停車、交通流量、旅運、通勤、OD、起訖點、道路、行人
**搜尋關鍵字（新北）：** 同上

**已在官方實作（跳過）：**
- 交通路況、公共停車場、今日施工資訊、最新交通事故月統計、監視器分佈
- YouBike 全系列（使用情況、週間群像、週末群像、見車率、見位率）
- 電動巴士比例、自行車道路統計資料
- 捷運系統全系列（人流趨勢、各線擁擠程度）

**步驟：**

### Step 1.1：開啟 data.taipei 並搜尋「公車」
```bash
playwright-cli open "https://data.taipei/dataset?q=公車&sort=viewCount"
playwright-cli screenshot --filename=research-transport-bus-taipei.png
```
記錄所有出現的資料集名稱與 URL（至少前3頁）。

### Step 1.2：搜尋其他通勤關鍵字（data.taipei）
依序搜尋：計程車、機車、停車場、交通流量、通勤、步行、行人
每次記錄資料集名稱 + URL。

### Step 1.3：搜尋 data.ntpc.gov.tw（新北）
```bash
playwright-cli open "https://data.ntpc.gov.tw/datasets?q=公車"
```
依序搜尋同一批關鍵字。

### Step 1.4：篩選 + 深度記錄
對每個找到的候選資料集：
1. 排除官方已實作的
2. 開啟資料集頁面，查看欄位說明、格式、更新頻率
3. 按上述格式填寫深度記錄
4. 打星評分（主題契合度 + 地圖潛力 + 雙北支援）

### Step 1.5：儲存文件
寫入 `docs/open-data-research/theme-01-transport.md`（目標 10 個以上）

|------|-----------|------|------|------|------|------------|

### Step 7.2：確認競賽門檻

檢查精選名單是否滿足：
- [ ] 至少 4 組「雙北（臺北＋新北都有對應資料）」的候選組件
- [ ] 至少 1 組有地圖圖層的雙北候選組件
- [ ] 所有資料集的更新頻率合理（非十年前的靜態資料）
- [ ] 資料格式可被前端直接使用（CSV/JSON/API）

### Step 7.3：標記開發優先順序

根據評分與競賽需求，標記：
- 🔴 **必做**：評分最高 + 雙北 + 地圖
- 🟡 **建議做**：評分高但無地圖，或地圖但雙北支援不完整
- 🟢 **備用**：評分中等，作為替補

## 執行注意事項

1. **每個 Task 執行完立即儲存文件**，不要等全部做完再寫，避免資訊遺失。
2. **資料集 URL 必須是真實可點擊的連結**，不能憑印象填寫。
3. **跳過評分 ≤ ⭐⭐ 的資料集**（更新頻率超過1年且無地理資訊）。
4. **優先記錄有 API 的資料集**，純靜態 CSV 評分降一星。
5. **每個主題確保至少2個來自新北（data.ntpc.gov.tw）**，確保雙北組件可行。
